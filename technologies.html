Protocols

Four field protocols you can't avoid.

Pick the right one for the right layer; force-fit none of them.

BACnet
Open · ASHRAE 135 · since 1995

The default building-automation protocol.

Born in ASHRAE committee work in the late '80s. BACnet's object model, analog input, binary value, schedule, calendar, trend log, maps cleanly onto how HVAC actually thinks. MS/TP for the field bus; BACnet/IP for the supervisor; BACnet/SC for the cyber-conscious 2026 version that wraps everything in TLS.

Strengths: rich object model, broad vendor support, built-in scheduling and trending. Weaknesses: discoverability across subnets is hard; MS/TP is fragile beyond ~30 devices; security is bolted on, not native (BACnet/SC fixes this).

Modbus
Open · since 1979

The simple, ugly, ubiquitous register protocol.

Forty-seven years old and still everywhere, every chiller, every power meter, every VFD will speak Modbus to you if you ask nicely. The protocol itself is dead simple: read or write 16-bit registers over RS-485 or TCP. Documentation lives in the device manual; there is no self-describing object model.

Use Modbus when BACnet isn't available and you control both ends. Avoid it as a building-wide backbone, register maps are vendor-specific and brittle.

MQTT
Open · ISO/IEC 20922 · since 1999

Lightweight pub/sub for cloud egress.

Originally written by IBM for telemetry over flaky satellite links. The pub/sub model suits HVAC well, publish a temperature topic, anyone interested subscribes. With the Sparkplug-B payload spec on top, you get a coherent way to describe edge devices and their birth/death lifecycle.

Not a replacement for BACnet, runs alongside it, north of the supervisory layer. The most common path for getting BMS data to a cloud historian or analytics platform.

TCP ports1883 / 8883 (TLS)
PayloadBytes · JSON · Sparkplug-B
QoS levels0 / 1 / 2
PatternPublish · Subscribe
LonWorks
Open · since 1990

The other open protocol, graceful, declining.

Echelon's network technology, popular in the 1990s for lighting, elevators, and parts of HVAC. The Standard Network Variable Type (SNVT) model is genuinely well-designed. New installations are rare, but you'll meet it in retrofit work, particularly on older campuses and Class-A office towers from the late '90s.

When integrating, plan on a LonWorks-to-BACnet gateway. Pure-Lon ecosystems are no longer worth committing to for new projects.

Data schemas

Tagging is half the work.

A protocol moves bytes. A schema makes them legible across vendors, across buildings, and across the next decade.

Project Haystack

A tag-based metadata model. Each point gets a set of tags: discharge, air, temp, sensor, plus references to the equipment that owns it. Lightweight, easy to retrofit, and the de-facto standard for North American commercial work.

Brick Schema

An RDF/OWL ontology developed by a university consortium. More formal than Haystack, uses explicit class hierarchies and relationships (isPartOf, feeds, hasPoint). The right tool when you need to reason across a building's structure, not just query its points. 

@prefix bldg: <http://linden.org#> .
bldg:AHU3 a brick:AHU ;

  brick:hasPoint bldg:DAT_3 .
bldg:DAT_3 a brick:Discharge_Air_Temperature_Sensor ;
  brick:hasUnit unit:DEG_F .

OT cyber

A building that is integrated is a building that is connected.

Five non-negotiables. Everything beyond these is hardening; nothing below them is acceptable.

Network segmentation

Controls network is its own VLAN, with explicit firewall rules to anything beyond it. No flat L2 across IT and OT.

Signed firmware

Every controller firmware update is cryptographically signed by the vendor and verified before flashing. No "TFTP and pray."

Credentialed access

Per-person accounts on the supervisor and the edge gateway. No shared "admin/admin." Rotate on staff turnover.

Audit trail

Every setpoint write, every override, every login, logged, immutable, retained for two years minimum.

Patching cadence

Defined maintenance windows for OT patches. Test in a lab environment before pushing to production controllers.

Vendor remote access

Time-bounded, just-in-time access for vendor support, not a permanent VPN tunnel. Logged and witnessed.

Integration patterns

Three architectures we deploy most.

Each one solves a different ownership and operations situation.

A

Single-vendor stack

One vendor's controllers, supervisor, and graphics. Cleanest install, fastest deployment. Highest lock-in. Right for small-to-mid commercial when the owner doesn't plan to own the building for fifteen years.

B

Multi-vendor on BACnet

Different vendors per system (chillers, AHUs, lighting), unified through BACnet/IP at the supervisor. The default we recommend for institutional owners. Each system bid competitively; the integration is the engineering work.

C

Federated cloud HBMS

Existing BMS stays in place; an edge gateway publishes tagged data to a cloud historian that aggregates across multiple buildings. The path most portfolios actually take, incremental, no rip-and-replace.
